EduCloud

A redesign of the online learning & teaching experience for universities

Project Overview

What is EduCloud?

EduCloud is an online teaching tool designed to improve the user experience of online learning for both students and educators.

The online teaching tool aims to let students access, and educators upload learning materials in a comfortable and confusion-free experience.

My Team Contributions

  • Conducting user research
  • Working with groupmates to create personas and user journey maps
  • Creating the high-fidelity prototype

What's the problem?

Yorkshare (VLE) is an online platform for the University of York students to access their module details, such as course materials, links to video conferencing tools, assignment submission areas, and other important tools for online learning.

However, some users reflected that they find the platform frustrating to use and hope for improvements.

Design Approach

  • User research
  • Affinity mapping
  • Desk-based research

Research findings

Main purposes for using online teaching tools/the VLE

Students

  • To submit assessment work
  • To contact their teachers

Educators

What's the solution?

Design requirements are determined by previously mentioned findings.

Functional requirements

Main requirements

  • to provide a clear summary of learning and teaching activities by a week or by topic
  • to have a good sense of hierarchy with fewer unnecessary options on the homepage

Detailed requirements for specific pages

Main menu

  • should include up-to-date learning materials
  • should feature a 'most accessed' function to access options that are used most frequently by the user

Module details pages

  • should clearly indicate essential and suggested resources
  • should allow teachers to check students' state of access to each learning resource
  • should give learning resources such as lecture slides to users
  • should show a clear description of each module

Assessment submission area

  • should provide a clear button for submitting assignments

Login page

  • should ask for preferences for setting up 2-step verification (including the option to exclude it altogether if the user wishes)
  • should provide other log-in methods such as using fingerprint verification and face ID

Additional requirements

  • each user should be able to access a personal dashboard
  • should include a discussion forum for students to participate in the class and share ideas about learning by adding comments
  • should provide contact details of the teaching staff

>

Non-functional requirements

  • have a clear layout
  • easy to navigate
  • have a simplistic and well-organised design
  • label buttons and links should be labelled accurately
  • The 'assessment submission' button should be large enough for users to notice at first glance
  • List of courses should be organised in one main area to avoid clutter
  • Courses should be differentiated using different colours
